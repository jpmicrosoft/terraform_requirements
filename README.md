### Additional Tools
- [TerraCognita - Read from your existing cloud providers and generate IaC in Terraform](https://github.com/cycloidio/terracognita)
- [InfraMap - Reads .tfstate or HCL to generate a graph specific for each provider](https://github.com/cycloidio/inframap)
- [TerraCost - Cloud cost estimation for Terraform in the CLI](https://github.com/cycloidio/terracost)
- [Blast-Radius - tool for reasoning about Terraform dependency graphs with interactive visualizations](https://github.com/28mm/blast-radius)
- [Terragrunt - thin wrapper for Terraform that provides extra tools for keeping your Terraform configurations DRY, working with multiple Terraform modules, and managing remote state](https://github.com/gruntwork-io/terragrunt)

### Testing Tools
- [TFLint - Terraform linter focused on possible errors, best practices, etc](https://github.com/terraform-linters/tflint)
- [terraform-compliance - lightweight, security and compliance focused test framework against terraform to enable negative testing capability for your infrastructure-as-code](https://terraform-compliance.com/)
- [Terratest - A Go library that makes it easier to write automated tests for your infrastructure code. It provides a variety of helper functions and patterns for common infrastructure testing tasks.](https://github.com/gruntwork-io/terratest)
- [Kitchen-Terraform - Set of Test Kitchen plugins which enable a system to use Test Kitchen to converge a Terraform configuration and verify the resulting Terraform state with InSpec controls](https://github.com/newcontext-oss/kitchen-terraform>)
